# 烟草物流数据管理系统 — 第一版设计文档

## 一、项目概述

### 背景与痛点

物流中心管理人员（约8人）日常报表统计工作量大，当前使用Excel独立作业：
- 每次统计都是独立工作，不同统计需求相当于不同工作量
- 统计质量不高，数据准确性难以保证
- 数据没有历史沉淀，无法回溯和复用

### 核心目标

数据一次录入，多维查询复用。通过系统化的数据导入、校验和报表能力，替代手工Excel统计。

### 部署方式

内网局域网部署，所有组件离线运行，保障数据安全。

## 五、核心流程

### 5.1 数据导入 → 校验 → 入库

```
用户上传 Excel
    │
    ▼
选择/匹配模板 → 解析 Excel → 数据预览表格
    │
    ▼
校验引擎执行（逐行）
    ├── 格式校验（类型、必填）
    ├── 字典一致性（人员/车辆是否存在且在岗）→ 不通过 = BLOCK
    └── 数值范围校验（里程、油耗等）→ 不通过 = WARN
    │
    ▼
前端展示校验结果
    ├── 全部通过 → 一键入库
    ├── 有 WARN → 红色高亮，可填备注后强制入库
    └── 有 BLOCK → 必须修正，支持在线编辑单元格 → 重新校验该行
```

### 5.2 报表双模式

**固定报表**：管理员预设查询模板，用户选择 → 填日期范围等参数 → 出表 → 可导出Excel

**灵活查询**：用户通过界面选表、选字段、加筛选条件、选聚合方式 → 实时预览 → 可保存为模板 → 可导出Excel

### 5.3 校验策略

- **BLOCK（强阻断）**：关键字段校验不通过，如人员不存在、车辆已停用。必须修正后才能入库。
- **WARN（弱预警）**：数值类校验不通过，如里程偏大、油耗偏高。红色高亮显示，允许填写备注后强制入库。
- 所有强制入库操作记录审计日志。

## 十、关键开发规范

- **SQL安全**：动态报表使用参数化查询，禁止字符串拼接SQL
- **表达式安全**：后续计算引擎使用simpleeval，禁止eval/exec
- **文件上传**：限制xlsx/xls类型，大小≤50MB
- **数据库迁移**：所有表结构变更通过Alembic，不手动改库
- **API规范**：统一响应格式 `{code, message, data}`，统一错误处理
- **角色权限**：admin可管理配置，operator只可操作业务数据和查询报表
